## ADDED Requirements

### Requirement: 回复服务能够处理用户问题
系统 SHALL 提供一个回复服务，接收用户问题并返回占位回答或规则化回答。

#### Scenario: 执行基础占位回答
- **WHEN** 后端将用户问题提交给回复服务
- **THEN** 回复服务返回一个可用于前端展示的文本回答

### Requirement: 回复服务可以利用会话上下文
系统 SHALL 允许回复服务接收当前会话中的历史消息，作为生成当前回答的上下文。

#### Scenario: 回答追问问题
- **WHEN** 用户在已有会话中继续追问
- **THEN** 回复服务可以利用历史消息生成与当前会话相关的回答

### Requirement: 回复服务应支持未来模型能力扩展
系统 SHALL 以可扩展方式组织回复服务，使后续新增大模型、Agent 编排或领域工具时，不需要修改核心聊天契约。

#### Scenario: 增加未来智能能力
- **WHEN** 团队在后续变更中接入大模型、Agent 编排或新的分析工具
- **THEN** 回复服务可以演进为新的实现，而无需重做前端会话流程或核心问答 API
