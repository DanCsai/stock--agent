## Why

项目需要先落下一个明确的基金推荐助手 MVP，这样产品和研发可以从讨论阶段进入真正的实现阶段。优先搭建一条可用的端到端聊天链路，有助于团队先验证交互方式、后端接口契约以及会话历史能力，再逐步投入更复杂的推荐逻辑、模型集成和行情分析能力。

## What Changes

- 为基金助手新增一个 Web 聊天界面，包含会话面板与历史会话列表。
- 使用 Go 和 Hertz 新增后端接口，用于创建会话、加载历史消息以及提交用户问题。
- 提供一个最小可运行的回复服务，能够接收用户输入并返回占位回答或规则化回答。
- 持久化保存会话与消息，方便用户回看历史记录并继续上下文对话。
- 建立一个可扩展的整体架构，先支持基金场景，后续可以用相同的聊天与服务模式平滑扩展到股票场景。
- 第一版范围聚焦前端界面、消息流转和历史会话能力，复杂推荐策略、模型接入、行情数据工具以及更丰富的分析组件放到后续变更中实现。

## Capabilities

### New Capabilities
- `fund-assistant-chat`: 提供基于聊天的用户体验，支持创建会话、查看历史记录以及与基金助手进行消息交互。
- `fund-assistant-query-api`: 定义后端会话管理与问答请求的 API 契约。
- `fund-assistant-reply-service`: 提供最小可运行的回复服务，用于生成占位回答并支撑前后端联调。

### Modified Capabilities
- 无。

## Impact

- 受影响系统：前端聊天应用、Go 后端服务、回复服务以及会话存储模块。
- 新增 API：创建会话、会话列表查询、消息历史查询以及助手问答提交接口。
- 新增依赖：Hertz 用于 HTTP 服务。
- 为后续工作打下基础：后续可以在同一交互模型上继续扩展真实基金数据源、模型能力、推荐逻辑、板块分析以及股票能力。
