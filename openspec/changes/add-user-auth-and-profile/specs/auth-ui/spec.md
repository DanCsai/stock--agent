## ADDED Requirements

### Requirement: 前端提供注册页和登录页
系统 SHALL 为用户提供可访问的注册页和登录页。

#### Scenario: 首次访问系统
- **WHEN** 未登录用户打开应用
- **THEN** 系统展示登录页或引导用户进入注册流程

### Requirement: 登录后界面展示当前用户信息
系统 SHALL 在登录后的主界面展示当前用户的用户名和头像。

#### Scenario: 登录后进入聊天页
- **WHEN** 用户登录成功并进入聊天页面
- **THEN** 页面展示当前用户的用户名和头像信息

### Requirement: 前端支持头像更新入口
系统 SHALL 在登录后的界面中提供头像更新入口。

#### Scenario: 打开头像更新入口
- **WHEN** 已登录用户在聊天页点击头像相关入口
- **THEN** 页面允许用户进入头像更新流程

### Requirement: 聊天界面需要具备更好的可读性和视觉层次
系统 SHALL 优化聊天主界面的布局比例、消息气泡宽度和内容留白，避免消息框过大或界面过空。

#### Scenario: 展示用户与助手消息
- **WHEN** 用户和助手消息出现在聊天区
- **THEN** 单条消息气泡宽度保持在适合阅读的范围内，且用户信息、消息内容与会话列表之间具备清晰层次
