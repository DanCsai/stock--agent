## MODIFIED Requirements

### Requirement: 系统提供会话管理 API
后端 SHALL 提供创建会话、查询会话列表以及获取指定会话消息的 API，并基于当前登录用户进行数据隔离。

#### Scenario: 创建会话请求
- **WHEN** 已登录客户端调用创建会话接口
- **THEN** 后端返回成功响应，并携带归属于当前用户的新会话标识

#### Scenario: 查询会话列表请求
- **WHEN** 已登录客户端调用会话列表接口
- **THEN** 后端仅返回当前用户可见的历史会话数据

#### Scenario: 获取会话消息请求
- **WHEN** 已登录客户端请求某个属于当前用户的会话消息历史
- **THEN** 后端按时间顺序返回该会话的消息内容

### Requirement: 系统提供助手问答 API
后端 SHALL 提供一个接收会话标识和用户问题的 API，用于执行助手问答流程并返回回答结果，同时校验该会话属于当前登录用户。

#### Scenario: 成功执行助手问答
- **WHEN** 已登录客户端针对一个属于当前用户的会话提交合法问题
- **THEN** 后端返回占位回答或规则化回答，并持久化保存用户消息和助手回复

#### Scenario: 使用未知或无权限会话发起问答
- **WHEN** 客户端提交的会话标识不存在或不属于当前登录用户
- **THEN** 后端返回一个表示会话无效或无权限的错误

### Requirement: API 响应支持前端聊天渲染
后端 SHALL 返回足够完整的会话和消息数据，使前端无需额外复杂转换即可渲染登录后的聊天界面。

#### Scenario: 发送消息后渲染回答
- **WHEN** 前端收到助手问答接口的响应
- **THEN** 前端可以基于返回数据直接在当前用户的活动会话中渲染最新的助手消息
