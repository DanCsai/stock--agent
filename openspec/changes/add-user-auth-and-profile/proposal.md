## Why

当前基金助手 MVP 还没有用户体系，所有会话记录都保存在同一份全局数据中，只适合单人本地演示，不适合真实使用。为了让后续聊天记录、用户资料和更完整的产品体验具备可扩展基础，需要尽快补齐注册登录、用户资料和按用户隔离会话的能力。

## What Changes

- 为系统新增用户注册和登录能力，支持账号、密码和用户名。
- 为系统新增用户资料能力，包含默认头像、查看资料和更换头像。
- 将现有聊天会话从全局共享改为按用户隔离，确保每个用户只能访问自己的会话和消息。
- 为前端新增登录页、注册页和登录后用户状态展示区域。
- 调整聊天页面布局和视觉样式，使其能够展示当前用户头像、用户名以及更完整、更精致的产品界面信息。
- 为后续更复杂的用户管理、会话同步和多端登录能力预留结构。

## Capabilities

### New Capabilities
- `user-auth`: 提供用户注册、登录和登录态校验能力。
- `user-profile`: 提供用户基础资料、默认头像和头像更新能力。
- `user-scoped-chat`: 将聊天会话与消息绑定到具体用户，防止不同用户共享会话数据。
- `auth-ui`: 提供登录、注册和登录后用户信息展示界面。

### Modified Capabilities
- `fund-assistant-chat`: 聊天界面需要在登录后展示，并显示当前用户信息。
- `fund-assistant-query-api`: 问答与会话接口需要基于当前登录用户进行访问控制和数据隔离。

## Impact

- 受影响系统：前端聊天界面、注册登录界面、后端认证接口、用户资料存储以及聊天存储结构。
- 新增 API：注册、登录、获取当前用户资料、更新头像。
- 修改 API：现有会话与消息接口需要加入用户鉴权和访问范围限制。
- 新增数据：用户账号、密码摘要、用户名、头像地址以及用户与会话的关联关系。
- 新增基础设施：MySQL 用于存储用户、会话、消息和资料数据。
- 为后续工作打下基础：后续可以继续扩展会话同步、找回密码、用户设置以及更完整的多用户产品体验。
