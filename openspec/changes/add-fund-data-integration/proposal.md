## Why

当前系统已经具备登录、会话和聊天界面能力，但聊天内容仍然主要依赖占位回复，缺少真实的基金业务价值。为了让产品从“可用骨架”进入“可查询真实基金信息”的阶段，需要新增基金搜索、基本信息和近期走势查询能力，并为后续推荐与分析模块打下基础。

## What Changes

- 为系统新增基金搜索能力，支持基金代码精确查询、基金名称模糊查询以及聊天输入触发查询。
- 为系统新增基金详情能力，展示基金名称、代码、类型、基金公司、最新净值和日涨跌等基础信息。
- 为系统新增基金走势能力，支持按天、月、3个月、半年、1年切换查看真实走势数据。
- 在前端引入“聊天简短回复 + 基金信息卡片 + 走势图 + 推荐/分析占位”的组合展示方式。
- 在后端抽象独立的基金数据服务和基金数据提供方接口，使网页端和后续飞书接入都能复用同一套能力。
- 为推荐占位和基金分析占位预留结构，但不在本次变更中实现真正的 Agent 推荐与分析逻辑。

## Capabilities

### New Capabilities
- `fund-search`: 提供基金代码精确查找、基金名称模糊查找以及候选列表返回能力。
- `fund-profile`: 提供基金基础信息查询能力。
- `fund-trend`: 提供基金指定时间范围走势查询能力。
- `fund-chat-card-ui`: 提供聊天文本、基金卡片、走势图和推荐/分析占位组合展示能力。
- `fund-provider-abstraction`: 提供可被网页端和后续飞书机器人复用的基金数据服务抽象。

### Modified Capabilities
- `fund-assistant-query-api`: 问答接口需要能够触发基金搜索、详情和走势查询，并返回结构化结果。
- `fund-assistant-chat`: 聊天界面需要支持基金搜索候选、基金卡片和走势展示。

## Impact

- 受影响系统：前端聊天页面、后端问答接口、基金数据服务层、真实基金数据源接入层。
- 新增 API：基金搜索、基金详情、基金走势查询相关能力接口，或在现有聊天接口上扩展结构化返回内容。
- 新增依赖：真实基金数据源接入能力，以及前端走势展示所需的数据格式支持。
- 后续扩展基础：后续可以在同一基金数据服务之上继续接推荐逻辑、基金对比分析、板块分析以及飞书机器人入口。
